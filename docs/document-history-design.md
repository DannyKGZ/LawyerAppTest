# Страница истории документа — обоснование

**Экран:** `/documents/[id]` · компонент `timeline-view.tsx`

## Главный вопрос экрана

> **Что сейчас происходит с документом и кто за что отвечает?**

Вся раскладка подчинена ответу на этот вопрос за 5 секунд, без чтения «стены текста».

## Решение

1. **Сводка в одну строку** — текущая версия, число открытых правок, блокеры, просрочки, конфликты, принятые правки «ждут версии». Цвет — только точками, текст нейтральный.

2. **Версии — горизонтальная цепочка** `v1 → v2 → v3` с кнопкой скачивания. Не смешиваем версии с правками по датам (как в старом таймлайне).

3. **Плашка «Требует внимания»** — только если есть блокеры, просрочки или конфликты. Явно называет проблему и имена ответственных.

4. **Таблица правок с фиксированными колонками:** пункт · **настоял** · статус · уровень · срок. В списке нет диффа и аргументации — только то, что нужно для сканирования.

5. **Конфликт — отдельная группа** с красной подложкой и заголовком «Конфликт · п. 7.1…», две строки внутри. Не две похожие строки в общем списке.

6. **Детали по клику** — боковая панель (Sheet): было/стало, настоял vs оформил, аргументация, действия. Старый EventJournal и тёмные аккордеоны по датам убраны как шум.

7. **Append-only:** закрытые правки (отклонено, внесено) остаются в таблице с пониженной непрозрачностью, не скрываются.

## Что убрали из старой версии

- Группировку по датам с тёмным раскрывающимся блоком
- EventJournal на этой странице
- Инлайн-метаданные в каждой строке (тип, просрочка текстом) — заменены колонками и маркерами

## Проверка

- `/documents/d1` — конфликт по п. 7.1, просрочка (п. 4.1), блокеры, разные статусы
- `/documents/d5` — документ попроще, только закрытые правки
