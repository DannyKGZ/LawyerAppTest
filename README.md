# LawyerApp

Веб-приложение «история правок юридических документов с ответственностью».
Концепция, модель данных и спека — в командной вики `team-wiki/LawyerApp/`.

## Стек

Next.js 16 · React 19 · TypeScript · Tailwind CSS 4 · shadcn/ui (base-nova, baseColor neutral) · lucide-react · sonner.

## Запуск

```bash
npm install
npm run dev      # http://localhost:3000
# или прод
npm run build && npm run start
```

## Что внутри (пока на мок-данных)

- **Реестр договоров** (`/`) — поиск, фильтр по сумме с переключателем валют (₽/$/€/¥, окно ±шаг), фильтр по менеджеру и статусу, быстрые фильтры «Блокеры»/«Просрочка», сортировка (по №/дороже/дешевле), сводка портфеля.
- **Страница проекта** (`/projects/[id]`) — реквизиты + документы пакета.
- **Таймлайн документа** (`/documents/[id]`) — сворачиваемые дни со статусом дня, сайд-бай-сайд дифф на уровне пункта, **уровень решения** (можно самим / ответственный / блокер·командой), детект конфликтов, карточка-ЗРС с решением (для блокеров — «Эскалировать команде»).
- **Как видит Claude** (`/agent`) — демо agent-слоя (get_state / resolve_document / prepare_next_version).

## Структура

```
src/lib/types.ts        — доменные типы
src/lib/domain.ts       — справочники (статусы/уровни/валюты) + хелперы (money, dayStatus, …)
src/lib/fixtures.ts     — мок-данные за доменным слоем
src/components/*-view.tsx — экраны (registry / project / timeline / agent)
src/components/ui/*      — shadcn-компоненты
reference/prototype.html — исходный кликабельный HTML-прототип (референс дизайна)
```

Данные временно мок (`src/lib/fixtures.ts`). Бэкенд (API + БД) — зона Эдема; предлагаемый контракт типов/эндпоинтов — в вики (`api-contract.md`). Мок-слой устроен так, чтобы позже подменить его на реальный API-клиент без переделки экранов.
