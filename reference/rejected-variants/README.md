# Отклонённые варианты редизайна страницы документа

> ⚠️ Это **архив «как НЕ надо»**, а не рабочий код. Файлы лежат с расширением `.bak`,
> вне `src/`, чтобы не попадать в сборку и роутинг. Хранятся как референс при
> разработке новых версий страницы `/documents/[id]`.

**Дата:** 2026-06-30
**Контекст:** попытка переработать страницу истории документа (`/documents/[id]`),
которая воспринимается как перегруженная и тяжёлая для чтения.

## Что пробовали

Два варианта (+ общая плашка «защита юриста»), подключались переключателем видов:

| Файл | Вариант | Идея |
|---|---|---|
| `timeline-cards-view.tsx.bak` | **A — карточки** | каждая правка = карточка: инициатор-герой, инлайн-дифф «было→стало», раскрытие подробностей в карточке |
| `timeline-clauses-view.tsx.bak` | **C — по пунктам** | группировка по пункту договора: текущая формулировка + кто за неё отвечает + цепочка правок |
| `timeline-shared.tsx.bak` | общее | `ProtectionStrip` (плашка «защита юриста»), `ViewSwitcher`, `Attribution`, `DiffInline` |
| `cards.page.tsx.bak`, `clauses.page.tsx.bak` | роуты | `/documents/[id]/cards`, `/documents/[id]/clauses` |

## Почему отклонено (вердикт заказчика)

**Слишком нагруженно.** Много данных на экране, куча текста — информацию тяжело
воспринимать. Обе раскладки добавляли элементы (карточки с полным диффом и
атрибуцией, плашки, цепочки), вместо того чтобы **убирать** лишнее. Цель страницы —
с одного взгляда понять, *что происходит с документом и кто за что отвечает*, — не
достигалась: глаз тонет в тексте.

## Вывод на будущее

- Лаконичность важнее полноты на главном экране. Детали — в карточку-ЗРС по клику,
  а не всё сразу в списке.
- Не «дописывать» поверх перегруженного экрана новые блоки — сначала решить, что
  на странице лишнее.
- Прежде чем верстать — определить **один** главный вопрос экрана и подчинить
  раскладку ему.
